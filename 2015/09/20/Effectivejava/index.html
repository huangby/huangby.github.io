第一条、考虑用静态工厂代替构造器
1.静态工厂方法与构造器不同的第一大优势在于，他们有名称
2.静态工厂方法与构造器不同的第二大优势在于，不必在每次调用他们的时候都创建一个新的对象。

3.第三大优势在于，他们可以返回原返回类型的任何子类型的对象。
4.第四大优势在于，在创建参数化类型市里的时候，它们使代码变得简洁。
缺点：
1.类如果不含公有的或者受保护的构造器，就不能被子类化。
2.他们与其他的静态方法实际上没有任何区别。

第三条、用私有的构造器或者枚举类型强化singleton属性
（单元素的枚举类型已经成为实现singleton的最佳方法）
使用enum关键字来实现单例模式的好处是这样非常简洁，并且无偿地提供了序列化机制，绝对防止多次实例化，即使是在面对复杂的序列化或者反射攻击的时候，代码如下：

1. public enum t{
2. INSTANCE;
3. private String name=null;
4. public void setName(String s){
5. name=s;
6. }
7. public void print(){
8. System.out.println(name);
9. }
10. public static void main(String [] args)
11. {
12. t test=t.INSTANCE;
13. test.setName("huangby");
14. test.print();
15. t test1=t.INSTANCE;
16. test1.print();
17. }
18. }

结果如下：
huangby

第九条、覆盖equals时总要覆盖hashCode
计算hash值的简单方法：
1.把某个非零的常数值，比如17，保存在一个名为result的int类型的变量中。
2.对于对象中的每个关键域f（指equals方法中涉及的每个域），完成以下步骤：
a.为该域计算int类型的散列码c.
1)如果该域是boolean类型，则计算(f?1:0).
2)如果该域是byte,char,short或者int类型，则计算(int)f
3)如果该域是long类型，则计算(int)(f^(f>>>32)).
4)如果是double类型，则计算Double.doubleToLongBits(f),然后按照步骤3）计算。
5)如果该域是一个对象引用，并且该类的equals方法通过递归地调用equals的方式来比较域，则同样为这个域递归地调用hashCode。如果需要更复杂的比较，则为这个域计算一个“范式”，然后针对这个范式调用hashCode.如果这个域是null,则返回0（或者是一个常数，但是通常为0）
6)如果该域是一个数组，则要把每一个元素当做单独的域来处理。
b.按照下面的公式，把把a中计算的散列码值合并到result中。
result=31*result+c;
3.返回result.

第十一条、谨慎地覆盖clone()方法
如果要覆盖一个非final类的clone方法，则应该返回一个通用调用super.clone而得到的对象。
实际上，clone方法就是另外一个构造器；你必须确保他不会伤害到原始的对象，并确保正确地创建被克隆对象中的约束条件
例如如果类中存在数组，则需要显示的克隆数组。代码如下：

1. @override public stack clone(){
2. stack result =(stack)super.clone;
3. result.elements=elements.clone();
4. return result;
5. }
   这里有个知识点，数组名调用colne方法，则会递归调用对象的clone方法，并且返回相同的类型。
      clone架构与引用可变对象的final域的正常用法是不相兼容的。
      所以实现cloneable接口的类都应该用一个公有的方法覆盖clone，此方法首先调用super.clone，然后修在任何需要修在的域。
      另外一个实现对象拷贝的好办法是提供一个拷贝构造器或者拷贝工厂。
      例如：
1. public Yum(Yum yum);
      拷贝工厂是类似于拷贝构造器的静态工厂：
1. public static Yum newInstance(Yum yum);
