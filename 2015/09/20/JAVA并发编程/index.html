JAVA并发编程

  • JAVA
JAVA

1、同步同样还具有另外一个重要、微妙的方面：内存可见性
2、java中基本数据类型都是原子性的，除了double和 Long，如果需要double和Long是原子性的，需要在其前面添加volatile关键字或者用锁

3、当一个域声明为volatile类型之后，编译器与运行时会监视这个变量：他是共享的，而且对他的操作不会与其他的内存操作一起被重排序。volatile变量不会缓存在寄存器或者缓存在对其他处理器隐藏的地方，所以读一个volatile类型的变量时，总会返回由某一线程所写入的最新值。
4、加锁可以保证可见性与原子性；volatile变量只能保证可见性
5、只有满足了下面的所有的标准后，才能使用volatile变量：
1)写入变量时并不依赖变量的当前值；或者能够确保只有单一的线程修改变量的值
2)变量不需要与其他的状态变量共同参与不变约束
3)而且，访问变量时，没有其他的原因需要加锁
6、在发布一个对象时，千万不要把对象的私有引用也给暴露出来(通常发生在直接返回私有数据的引用)
class Test{
private String [] names ={“zhangsan”,”lisi”,”waner”,”dada”};
……
public String[] getNames(){
……
return names
}
}
上面的列子中，虽然names是私有的变量，但是却在getNames方法中把这个引用返回了，等于完全暴露了，其他对象能够修改。
所以千万不要这样做，这里可以使用拷贝的方式解决。
